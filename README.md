# 🥬 MilaCLI

Локальный консольный AI-агент «Мила» для разработки, работающий поверх
любого OpenAI-совместимого API — включая панель `https://aihub.071129.xyz`.

Два режима работы:

- **Code Mode** (по умолчанию, как Claude Code / Codex CLI) — доступ только
  к файлам текущего проекта и терминалу.
- **Agent Mode** (`mila --agent`) — плюс Мила "видит" экран (скриншот →
  vision), реально управляет мышью и клавиатурой (RPA) и ищет в интернете
  настоящими HTTP-запросами.

## Установка

### Обычная (сам вводишь свой ключ)

```bash
cd milacli
npm install
npm link      # регистрирует глобальную команду `mila`
mila config   # свой API-ключ, effort и т.д. через интерактивный мастер
```

### Одной командой (общий ключ, для раздачи) — Termux или ПК

```bash
cd milacli
bash install.sh
```

Скрипт сам определяет платформу:

- **На ПК** (Linux/macOS/Windows-WSL) — спрашивает, ставить ли Agent Mode
  (реальные клики мышью/клавиатурой + скриншот). Если да — тянет нативные
  пакеты `@nut-tree-fork/nut-js` и `screenshot-desktop`.
- **На Termux (Android)** — вопрос не задаётся: Agent Mode с физическими
  кликами на Android не работает (ограничение платформы, не баг),
  автоматически ставится только Code Mode.

Дальше в обоих случаях: регистрирует `mila`, спрашивает уровень **effort**
(low/medium/high) и сразу создаёт готовый профиль с уже зашитым ключом —
без интерактивного `mila config`.

⚠️ **Важно**: `install.sh` содержит реальный API-ключ в открытом виде.
Любой человек, открывший этот файл, может увидеть и использовать ключ —
не только через MilaCLI, а вообще где угодно (curl, свои скрипты и т.д.).
Раздавай архив только тем, кому доверяешь, и следи за расходом на панели
`aihub.071129.xyz` — общий ключ без контроля может быть исчерпан очень быстро.

⚠️ **На Termux/Android** пакет `@nut-tree-fork/nut-js` (реальные клики
мышью/клавиатурой) не устанавливается вообще — Android не даёт того же
доступа к системным событиям ввода, что desktop-ОС. Это ограничение
платформы, не баг. `npm install` теперь не падает из-за этого (пакет
вынесен в `optionalDependencies`): Code Mode работает полностью, а в
Agent Mode `take_screenshot`/`get_screen_size`/`web_search`/`web_fetch`
тоже работают — только `mouse_click`/`keyboard_type`/`keyboard_press`
вернут понятную ошибку вместо клика.

## Быстрый старт

```bash
mila config          # интерактивная настройка профиля + пинг-тест соединения
mila                 # Code Mode: файлы + терминал
mila --agent         # Agent Mode: + экран, мышь/клавиатура, веб
mila run "создай express-сервер с health-check эндпоинтом"
mila run --agent "открой браузер и найди курс доллара"
```

При `mila config` можно сразу принять значения по умолчанию — они уже
указывают на `https://aihub.071129.xyz/v1` и модель `step-3.5-flash-2603`.
Нужно будет ввести только свой API-ключ.

При запуске терминал очищается и MilaCLI рисует себя отдельной рамкой
(баннер + boxen), похоже на отдельное "окно" поверх обычной консоли — с
чёткой пометкой текущего режима (CODE MODE / AGENT MODE).

## Команды

| Команда                        | Описание                                                        |
|----------------------------------|--------------------------------------------------------------------|
| `mila config`                   | Пошаговая настройка профиля API + диагностика связи                |
| `mila profiles`                 | Список сохранённых профилей                                         |
| `mila use <name>`               | Переключить активный профиль                                        |
| `mila run "<prompt>"`           | Одноразовый запуск агента с задачей (Code Mode)                     |
| `mila run --agent "<prompt>"`   | Одноразовый запуск в Agent Mode                                     |
| `mila`                          | Интерактивный REPL-диалог, Code Mode                                 |
| `mila --agent`                  | Интерактивный REPL-диалог, Agent Mode                                |

Внутри REPL: `/exit` — выход, `/tokens` — статистика расхода, `/clear` — сброс контекста.

## Инструменты

**Code Mode** (`src/tools.js`): `read_file`, `list_dir`, `write_file`,
`edit_file`, `delete_file`, `run_command`, `search_code`.

**Agent Mode добавляет** (`src/agent-tools.js`):

| Инструмент         | Что делает                                             | Риск   |
|----------------------|------------------------------------------------------------|--------|
| `take_screenshot`   | Снимок экрана, передаётся модели как изображение (vision) | низкий |
| `get_screen_size`   | Разрешение экрана в пикселях                                | низкий |
| `mouse_move`        | Перемещает курсор в координаты                              | средний|
| `mouse_click`       | Реальный клик мышью (лево/право/центр, одиночный/двойной)   | высокий|
| `keyboard_type`     | Реальный ввод текста с клавиатуры                            | высокий|
| `keyboard_press`    | Реальное нажатие комбинаций клавиш (Enter, Ctrl+C и т.д.)   | высокий|
| `web_search`        | Настоящий поиск в интернете (DuckDuckGo HTML, без API-ключа) | низкий |
| `web_fetch`         | Загрузка и очистка текста веб-страницы по URL                 | средний|

## Архитектура

```
src/
  config.js        — профили и permissions в ~/.milacli/*.json
  provider.js       — HTTP/SSE клиент OpenAI-совместимого API (поддерживает vision-контент)
  tools.js          — Code Mode: read/write/edit/delete файлов, run_command, search_code
  agent-tools.js    — Agent Mode: screenshot, mouse/keyboard (RPA), web_search/web_fetch
  permissions.js    — интерактивные запросы подтверждения (sandbox, risk levels)
  tokens.js         — счётчик токенов в реальном времени
  agent.js          — Reasoning Loop + Tool-Calling Loop + Auto-Heal, роутинг между наборами инструментов
  index.js          — CLI (commander), флаг --agent, очистка терминала, "окно"-баннер
```

## Безопасность

Каждое действие инструмента проходит через двухэтапную проверку прав:
уровень риска (`low` / `medium` / `high`) и интерактивный запрос
подтверждения — «один раз», «в этой сессии», «всегда» или
«отклонить/прервать». Флаг `-y/--yes` подтверждает только low/medium —
`delete_file`, `run_command`, `mouse_click`, `keyboard_type`,
`keyboard_press` всегда требуют явного подтверждения, даже с `-y`.

⚠️ Agent Mode физически управляет твоим компьютером — реальные клики и
реальный ввод текста, отменить действие после факта нельзя. Не давай
«разрешать всегда» для кликов/ввода без разбора, особенно на рабочем
компьютере с открытыми личными аккаунтами.

Скриншоты могут требовать системных разрешений:
- **macOS**: Настройки → Конфиденциальность → Запись экрана
- **Linux**: зависит от X11/Wayland и наличия `scrot`/аналогов
- **Windows**: обычно работает без дополнительных прав
